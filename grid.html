Grid

This grid provides the building blocks for a responsive application. Below is an overview of the different parts of the grid and how they work together.

Structure

The grid consists of containers, rows and columns.

  • Containers set the page margins and can contain one or more rows.
  • Rows hold one or more columns.
  • Columns are used create the layout of a page or component.

Mobile First

This grid system is mobile-first, which means that by default the layouts established for mobile will apply to all other screen sizes. From there, you can specify column widths for the larger screen sizes, if desired.

Fixed vs. fluid container

The container class applies a fixed width per breakpoint to the container. Use container-fluid to enable fluid containers.

Breakpoint-specific widths

Set specific widths for each breakpoint using col-{screen-size}-{columns}.

xs-10, sm-8, md-6, lg-4
<div class="container-fluid">
    <div class="row show-grid">
        <div class="col-xs-10 col-sm-8 col-md-6 col-lg-4">xs-10, sm-8, md-6, lg-4</div>
    </div>
</div>

Column offsets

Offsets are handy for shifting or centering columns in your layouts. Just like column widths themselves, offsets are specific to each breakpoint.

Nesting

You can nest a row inside of a column, which is handy for creating component layouts. Note that each time you nest a row, you get a new set of 12 columns relative to the parent column you're nesting within. So a row nested inside of a 6-column parent will have columns that are half as wide as its parent.
