Body copy

Body copy should be used to communicate detailed information to users.

Aenean vestibulum metus pellentesque elit faucibus, placerat gravida nisl consequat. Maecenas dignissim, mauris id feugiat fringilla, nibh arcu interdum turpis, eu ornare orci velit quis nisi. Donec pulvinar tempus urna eget placerat.

<p>Aenean vestibulum metus pellentesque elit faucibus, placerat gravida nisl consequat. Maecenas dignissim, mauris id feugiat fringilla, nibh arcu interdum turpis, eu ornare orci velit quis nisi. Donec pulvinar tempus urna eget placerat.</p>

Lead copy

Lead copy is useful for subheadings or to call attention to important high-level information.

Lorem ipsum dolor sit amet, consectetur adipiscing elit. Etiam pellentesque mollis metus, eget suscipit metus.

<p class="lead">Lorem ipsum dolor sit amet, consectetur adipiscing elit.  Etiam pellentesque mollis metus, eget suscipit metus.</p>

Buttons

Options

Each button style represents a different type of action.

<button type="button" class="btn btn-primary">Call to action</button>
<button type="button" class="btn btn-info">Learn more</button>
<button type="button" class="btn btn-default">Go back</button>
<button type="button" class="btn btn-danger">Delete</button>
<button type="button" class="btn btn-link">Link</button>

Ghost variants

Some buttons offer outlined or "ghost" variants.

<button type="button" class="btn btn-info btn-ghost">Learn more</button>
<button type="button" class="btn btn-default btn-ghost">Go back</button>

Sizes

Various button sizes are available for different applications.

<button type="button" class="btn btn-info btn-lg">Large</button>
<button type="button" class="btn btn-info">Regular</button>
<button type="button" class="btn btn-info btn-sm">Small</button>
<button type="button" class="btn btn-info btn-xs">x-small</button>

Block level

A block level style is available to make buttons full-width within grid columns or other containers.

<div class="row">
    <div class="col-sm-4">
        <button type="button" class="btn btn-primary btn-block">Submit</button>
        <button type="button" class="btn btn-default btn-block">Go back</button>
    </div>
</div>

Disabled state

Disabled buttons are given a lighter background and are not selectable.

<button type="button" class="btn btn-primary" disabled>Call to action</button>
<button type="button" class="btn btn-info" disabled>Learn more</button>
<button type="button" class="btn btn-default" disabled>Go back</button>
<button type="button" class="btn btn-danger" disabled>Delete</button>
<button type="button" class="btn btn-link" disabled>Link</button>

Forms

Basic form

Basic forms include input fields with a label for each field and a submit button.

<form>
    <div class="form-group">
        <label for="exampleInputEmail1">Email address</label>
        <input type="email" class="form-control" id="exampleInputEmail1" placeholder="Email">
    </div>
    <div class="form-group">
        <label for="exampleInputPassword1">Password</label>
        <input type="password" class="form-control" id="exampleInputPassword1" placeholder="Password">
    </div>
    <button type="submit" class="btn btn-primary">Submit</button>
</form>

Inline form

Inline forms are handy when you want the user to navigate the fields from left-to-right.

<form class="form-inline">
    <div class="form-group">
        <label for="exampleInputName2">Name</label>
        <input type="text" class="form-control" id="exampleInputName2" placeholder="Jane Doe">
    </div>
    <div class="form-group">
        <label for="exampleInputEmail2">Email</label>
        <input type="email" class="form-control" id="exampleInputEmail2" placeholder="jane.doe@example.com">
    </div>
    <button type="submit" class="btn btn-primary">Sign up</button>
</form>

Horizontal form

Horizontal forms take advantage of wide areas by placing labels to the left of their corresponding input fields.

<form class="form-horizontal">
    <div class="form-group">
        <label for="inputEmail3" class="col-sm-2 control-label">Email</label>
        <div class="col-sm-10">
            <input type="email" class="form-control" id="inputEmail3" placeholder="Email">
        </div>
    </div>
    <div class="form-group">
        <label for="inputPassword3" class="col-sm-2 control-label">Password</label>
        <div class="col-sm-10">
            <input type="password" class="form-control" id="inputPassword3" placeholder="Password">
        </div>
    </div>
    <div class="form-group">
        <div class="col-sm-offset-2 col-sm-10">
            <button type="submit" class="btn btn-primary">Sign in</button>
        </div>
    </div>
</form>

Icons

Filled icons

The default option for each icon is the filled variant.

<span class="icon icon-arrow-up"></span>
<span class="icon icon-plus"></span>
<span class="icon icon-success"></span>
<span class="icon icon-caution"></span>

Outlined icons

Add the class icon-outline to enable the outlined variant.

<span class="icon icon-outline icon-arrow-up"></span>
<span class="icon icon-outline icon-plus"></span>
<span class="icon icon-outline icon-success"></span>
<span class="icon icon-outline icon-caution"></span>
